ADK SIF Compliance Testing

SIF Versions

  • The compliance test harness doesn’t allow the agent to register with versions prior to SIF 2.0r1. Before you start testing, set a property to tell the ADK to register with a custom set of SIF Versions:
    <property name=”adk.provisioning.overrideSifVersions” value=”2.0r1,2.1,2.*” />
  • The ADK SIF Version should be set to the version the compliance test harness is testing. (E.g. “2.0r1” for the 2.0r1 test harness)

SSL settings

  • Make sure your CSQ matches the SSL settings below
  • Make sure your agent creates a certificate with a DSA key-pair. Most of our agents, by default, create one with an RSA key-pair. The test harness requires DSA in one of it’s tests because DSA is a required cipher for TLS 1.0.

Section 5: Implementation Dependent Behavior

Here is a preliminary set of claims that should be supported by all ADK agents.

Characterization of the product’s implementation-dependent behavior

Transport Support

Check both SIF HTTPS and SIF HTTP

Mode Support

Check both Push and Pull

HTTPS Support

-Which SSL versions does your application support for initiating connections to the ZIS?-

Don’t Check: TLS 1.0 and SSL 3.0
Check: TLS 1.0 negotiated using an SSL 2.0 helloSSL 3.0 negotiated using an SSL 2.0 hello

-If your application supports Push mode, which SSL versions can your agent accept when receiving incoming connections from the ZIS?-

Check all four

Security

SIF Authentication Level Support

-Is your agent capable of returning client certificates (authentication level 1-3)?-

Yes

-If your agent ever explicitly requests authentication levels 1-3 of other agents at delivery time, which authentication levels might be explicitly requested?-

1,2,3

-If your agent authenticates server certificates returned by the ZIS, which authentication levels might be required?-

1,2,3

-If your agent supports Push mode and your agent ever requires a client certificate from the ZIS, which authentication levels might be required?-

1,2,3

SIF Encryption Level Support

Encryption Level

-Which encryption levels is your agent capable of supporting in SIF HTTPS?-

1,2,3,4 (don’t check 0)

-If your agent ever explicitly requests encryption levels of other agents at delivery time, which encryption levels might be required?-

1,2,3,4 (don’t check 0)

-Which encryption levels might your agent require from a ZIS in SIF HTTPS?-

1,2,3,4 (don’t check 0)

Message Validation

Is your application capable of performing XML validation on incoming messages?

The ADK doesn’t support XML validation out of the box, but it does do validation that the XML at least matches its internal metadata. You can choose 'Yes' or 'No' here.

Error Handling

-If a SIF_Ack is returned with any type of error category other than a transport error, does the application stop trying to send the message (delete or inactivate the message) in order to avoid a potential deadlock condition?-

Yes

Buffer Size

-Maximum SIF_MaxBufferSize (in bytes):- Suggested: 1048576

Unicode Support

-Does your agent support Unicode (e.g. it does not map back and forth between Unicode and a local character set)?-

Yes
