Release 2.2.0

New Features & Enhancements

6 new features and enhancements are included in this release.
Mappings - Map a real database value to null

The ADK now supports suppressing a valueset translation if there is no match. (Ticket #198. Addressed in 2.2.0)
Mappings: restrict values to a valueset

The ADK now supports restricting the set of mapped values to a valueset. (Ticket #201. Addressed in 2.2.0)
ADK ValueSet class does not have a way to remove app default or sif default values that have been set

The ValueSet Application default and SIF default values can now be cleared by calling ValueSet.AppDefault and ValueSet.SifDefault and passing in null as the default value. (Ticket #2125. Addressed in 2.2.0)
Add the capability to specify the packet # and SIF_MorePackets flag in SIFResponseSender

The SIFResponseSender class now has APIs that allow the SIF_MorePackets and SIF_PacketNumber values to be set. (Ticket #2458. Addressed in 2.2.0)
Add the capability to specify the packet # and SIF_MorePackets flag in SIFResponseSender

The SIFResponseSender class now has APIs that allow the SIF_MorePackets and SIF_PacketNumber values to be set. (Ticket #2458. Addressed in 2.2.0)
Add 2.* capability to the SIF Version class

The ADK now properly parses and uses 2.* in XML parsing. (Ticket #2977. Addressed in 2.2.0)

Fixes

9 defects are addressed by this release.
SIF_Register/SIF_Version is not being rendered with the "default" version of the agent being specified first

The ADK now renders the default version supported by the agent first, which has some significance to some zone integration servers. (Ticket #748. Addressed in 2.2.0)
getElementOrAttribute uses version-independent tag names for comparision against XPath

The ADK now evaluates XPaths using version-dependent tag name checking. (Ticket #1554. Addressed in 2.2.0)
StudentSnapshot object has rendering issues with ordering of elements

The StudentSnapshot GradeLevel element was being rendered in the wrong sequence order in SIF 1.5r1. This has been fixed in ADK 2.2.0. (Ticket #2101. Addressed in 2.2.0)
Change the StringMapAdaptor class to default converted values to NULL if the source field is an empty stirng and the destination type is not a string

Changed the default mapping behavior for non-string data types. If a value being mapped is a string and it is impty or contains only spaces and needs to be converted to a non-string type, such as numeric, date, or duration, it is mapped as 'NULL'. (Ticket #2356. Addressed in 2.2.0)
ADK sometimes renders elements in StudentSchoolEnrollment out of order in SIF 2.x

The ADK now properly orders the rendering of StudentSchoolEnrollment child elements. (Ticket #2383. Addressed in 2.2.0)
SchoolCourseInfo does not publish SubjectArea in SIF 1.x

The ADK now properly renders SchoolCourseInfo/SubjectArea in SIF 1.1 and SIF 1.5r1. (Ticket #2566. Addressed in 2.2.0)
Definition of StudentPersonal/StateProvinceId is incorrect

StudentPersonal/StateProvinceId is not correctly typed as a string, not as a StatePrCode SIFEnum. (Ticket #2694. Addressed in 2.2.0)
Add support for mapping to and from SIF 1.x

The ADK for .NET now supports mapping to and from SIF 1.1 and 1.5r1, in addition to supportin SIF 2.0r1, 2.1, and 2.2. (Ticket #2836. Addressed in 2.2.0)
Add support for mapping to and from SIF 1.x

The ADK now supports mapping to and from SIF 1.x. (Ticket #2836. Addressed in 2.2.0)

May contain other minor bug fixes not listed above.


Release 2.0.2

New Features & Enhancements

5 new features and enhancements are included in this release.
The .NET ADK should maintain the order of repeatable elements added to a SIFDataObject t

The ADK now renders like elements (repeatable elements) in the same order in which they were added to the parent. (Ticket #2826. Addressed in 2.0.2)
Add support for parsing objects, even if the datatypes cannot be parsed into their proper values

If the AgentProperty "adk.messaging.strictTypeParsing" is set to 'false' (which is default), the ADK will continue parsing a SIF_Message even if it cannot parse an XML element or attribute value into the appropriate data type. The resulting field will be set to NULL, and the parsing error will be logged to the zone log. (Ticket #2828. Addressed in 2.0.2)
Add support for the SIF 2.2 Data Model

This release includes support for SIF Objects and elements added in the SIF 2.2 specification. (Ticket #2839. Addressed in 2.0.2)
Add support for the SIF UK 1.1 Data Model

The SIFWorks ADK has been updated to support the SIF UK 1.1 release. (Ticket #2845. Addressed in 2.0.2)
Update log4net to 1.2.10

The ADK now uses the 1.2.10 release of log4net. (Ticket #2997. Addressed in 2.0.2)

Fixes

6 defects are addressed by this release.
ADKException.setRetry is ignored while processing a SIF_Response

If the AdkException.Retry property is set to TRUE, the ADK will return an error SIF_Ack to the ZIS, with a specific code that signals the message should remain in the agent's queue. (Ticket #260. Addressed in 2.0.2)
Add support for xsi:nill

The ADK now renders elements as xs:nill if the element value has been set to a null value using SetField(). In addition, the ADK will parse elements with a value of xs:nill value. Note that using the property accessor for the fields will return null if the element is missing or the element was included, but has a value of xs:nill. (Ticket #2051. Addressed in 2.0.2)
Change the ADK to match the recent SIF change of StudentAttendanceSummary from int to decimal

The datatypes for the attendance value elements in StudentAttendanceSummary have changed from int to decimal to match a recent change in the SIF specification. (Ticket #2825. Addressed in 2.0.2)
ADK returns empty SIF_Response packet if the SIF_Request contains SIF_Element restrictions for elements that are not contained in the response data

The ADK now properly renders the root element and attributes of a SIF Object if the SIF_Request contains SIF_Element restrictions that specify elements not contained in the object. (Ticket #2849. Addressed in 2.0.2)
Problem with Query.evaluate

This release fixes an issue with Query.Evaluate. Previously, Query.Evaluate() would always return true if the SIF_Query had a SIF_ConditionGroup with a 'Type' of 'Or'. This has been fixed and Or query conditions are evaluated properly to only return true if one of the conditions is true. (Ticket #2922. Addressed in 2.0.2)
Issue with Unique Property Number

The datatypes for UniquePropertyNumber and UniqueStreetNumber have been changed to Int64 (long). (Ticket #2961. Addressed in 2.0.2)

May contain other minor bug fixes not listed above.

The ADK mappings and xpath support are based upon true XPath expressions, and can easily identify repeatable elements by ordinal or key. (Ticket #1499. Addressed in 2.0.1)

The ADK Developers license that ships with the .NET ADK is valid through July, 2009. (Ticket #1545. Addressed in 2.0.1)
[.NET] (UK) Missing support for some elements on LearnerPersonal

The LearnerPersonal l -ULN -ConnexionsAgreement -GiftedTalented -UniformAllowance elements are now supported. (Ticket #1794. Addressed in 2.0.1)
Element.setChanged() does not work as designed

Element.setChanged() now properly works as designed. When calling setChanged() or setChanged(true), all child elements are marked as change, as well as immediate parents. When calling setChanged(false), all children are marked as not changed. (Ticket #1798. Addressed in 2.0.1)
SIFDataObject.SIFExtendedElements property always returns NULL

The SIFExtendedElements property on SIFDataObject now returns an empty array if no elements are set. (Ticket #1883. Addressed in 2.0.1)
The Query class does not render properly to XML if the root condition groups has condition groups

The ADK Query class now properly renders itself to XML, even with complex Query condition groups. (Ticket #1917. Addressed in 2.0.1)
Add support for "2.*" in SIF_Requests

When the ADK receives a SIF_Request with a SIF_Version of "2.*", it automatically translates that to the latest version of SIF supported by the ADK. Currently, it is translated to the same value as ADK.getSIFVersion to please the test harness. In the future, and once SIF certification agrees, it will return the latest version of SIF supported by the ADK. (Ticket #2006. Addressed in 2.0.1)
UK Edition (.NET) needs updated ZIS license keys

The .NET ADK now builds with the most current developer's license for the ZIS. (Ticket #2178. Addressed in 2.0.1)
(UK) Data model Issues with the UK 2.1 ADK

The ADK fully supports the UK draft 1.1 data model as of the November 30th, 2007 release for developers. (Ticket #2303. Addressed in 2.0.1)
ADK should not include a SIF_Protocol element when registering in pull mode

The ADK now properly registers in pull mode without a SIF_Protocol element. (Ticket #2462. Addressed in 2.0.1)
ADK improperly sends URLS in decoded format

The ADK now properly encodes URLS when connecting to the ZIS. (Ticket #2468. Addressed in 2.0.1)
ADK sends improper SIF_Version element in custom SIF_Register

When using the adk.provisioning.overrideSifVersions property, the ADK now cleans out any spaces so that invalid SIF_Version elements are not created. (Ticket #2469. Addressed in 2.0.1)
Add support for requesting data using 2.*

The ADK automatically by default requests data using 2.* when running in SIF 2.0 or higher. If a specific SIF_Request/SIF_Version is desired, it can be set programmatically on the query object or configured at runtime by applying policy elements to the agent configuration file. (Ticket #2494. Addressed in 2.0.1)
Add support for deeply nested SIF_Element restrictions in SIF_Query

The ADK now properly handles SIF_Request with deeply nested SIF_Element restrictions. The ADK automatically handles returning just the elements that were requested in the SIF_Request. (Ticket #2495. Addressed in 2.0.1)
Query.evaluateCondition is not properly comparing SIF 2 XSD Datatypes

Query.evaluateCondition() now properly converts all data values to their SIF 1.x or 2.x data format before comparing. (Ticket #2496. Addressed in 2.0.1)
ADK Should return proper error category and code when it receives a SIF_Message in an unsupported SIF_Context

The ADK returns an error category of 12 and a code of 4 when it receives a SIF_Request in an unsupported SIF Context. (Ticket #2497. Addressed in 2.0.1)
ADK Should return an appropriate SIF Error Category and code when it receives a SIF_Request with an unsupported buffer size

The ADK returns an error category of 8 and a code of 8 when it receives a SIF_Request with an unsupported SIF_MaxBufferSize. (Ticket #2498. Addressed in 2.0.1)
ADK Should return an appropriate SIF Error Category and Code when it receives an unsupported request for SIF_ExtendedQuery

The ADK now properly returns an error category of 8 and a code of 15. (Ticket #2499. Addressed in 2.0.1)
IMessagingListener.OnMessageProcessed delivers SIF_Request Messages with a message type of SIF_Response when it should be SIF_Request

SIF_Requests are now appropriately marked in IMessagingListener.OnMessageProcessed. (Ticket #2503. Addressed in 2.0.1)
ADK creates invalid SIF_Ack when Sleeping

The ADK now properly uses the SIF_Status/SIF_Desc to report the "Receiver is sleeping" message. (Ticket #2513. Addressed in 2.0.1)

The ADK no longer has errors at startup about the ObjectGraph being null. (Ticket #2514. Addressed in 2.0.1)
